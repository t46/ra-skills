# 研究広報支援

## What This Does

研究成果を社会・学内外へ伝えるために、プレスリリース、研究紹介文、Web掲載文、取材回答メモ、英文概要、SNS向け短文の初稿を作る。広報担当、PI、共同機関、知財、契約、倫理、FAの確認を前提に、公開前チェックを含む広報フローを整理する。

- **対象ユーザー**: URA、広報担当、PI、研究支援担当者、プロジェクトマネージャー
- **解決する課題**: 研究成果の意義が専門外に伝わらず、公開前確認も漏れやすい
- **期待される効果**: 分かりやすい広報文、確認済みの公開範囲、共同機関との合意を整える

## Required Inputs

- **成果情報**: 論文、発表、受賞、採択、データ、ソフトウェア、イベントなど
- **研究の要点**: 何を明らかにしたか、何が新しいか、社会的意義、限界
- **公開予定**: 論文公開日、 embargo、イベント日、Web掲載希望日
- **確認先**: PI、共著者、共同機関、広報、知財、契約、倫理、FA
- **素材**（任意）: 論文、図、写真、DOI、研究者プロフィール、既存説明文
- **対象読者**（任意）: 一般、記者、学内、企業、学生、政策関係者

## What It Produces

1. **研究広報ブリーフ**: 一文要約、背景、新規性、意義、注意点
2. **プレスリリース初稿**: タイトル、リード、本文、研究者コメント、問い合わせ先
3. **Web掲載文・SNS文案**: 短く読める紹介文
4. **公開前確認リスト**: 知財、契約、個人情報、倫理、共同機関、embargoの確認
5. **取材対応メモ**: 想定質問、答えてよい範囲、避ける表現

## 手順

### 1. 広報できる状態か確認する

公開前に、知財・契約・倫理・共同機関・出版社 embargo を確認する。未確認なら「公開不可」ではなく「要確認」とする。

```markdown
公開前確認:
- 論文公開日/embargo:
- 知財確認:
- 共同機関確認:
- 契約上の制限:
- 個人情報・倫理:
- FA表記・謝辞:
```

### 2. 研究の要点を専門外向けに整理する

```markdown
- 何をしたか:
- 何が分かったか:
- 何が新しいか:
- なぜ重要か:
- どこまで言えるか:
- 誤解されやすい点:
```

誇張、断定、医療・政策・社会実装上の過剰な示唆は避ける。

### 3. 広報ブリーフを作る

| 項目 | 内容 |
|------|------|
| 一文要約 | 専門外に伝わる1文 |
| 背景 | なぜこの研究が必要か |
| 新規性 | 従来との差分 |
| 成果 | 分かったこと、作ったもの |
| 意義 | 研究・社会・政策・産業への示唆 |
| 限界 | まだ言えないこと |
| 確認事項 | 公開前に見るべき点 |

### 4. プレスリリース初稿を作る

```markdown
# タイトル

## ポイント
- {ポイント1}
- {ポイント2}
- {ポイント3}

## 概要
{専門外向けのリード}

## 背景

## 研究内容

## 今後の展望

## 論文・成果情報

## 研究費・謝辞

## 問い合わせ先
```

### 5. 確認フローを作る

```markdown
| 確認者 | 見る観点 | 期限 | 状態 |
|--------|----------|------|------|
```

共同機関がある場合は、公開日時、機関名表記、問い合わせ先、図版使用可否を必ず確認する。

## 判断基準

| 判断項目 | 見るポイント | 対応 |
|----------|--------------|------|
| 正確性 | 論文・成果の範囲を超えていないか | PI確認へ回す |
| 公開可否 | 知財・契約・embargo・倫理に触れないか | 専門部署確認 |
| 分かりやすさ | 専門外に一文で伝わるか | 背景から書く |
| 表記 | 研究費、所属、氏名、課題番号が正しいか | 公式表記を確認 |
| 共同機関 | 公開日時・文面に合意があるか | 確認表に残す |

## よくあるミスと対策

| ミス | 対策 |
|------|------|
| 成果を大きく見せすぎる | 限界と未解明点を入れる |
| 知財確認前に発明内容を公開する | `ip-disclosure-triage` に渡す |
| 共同機関の確認を忘れる | 確認者表に共同機関を入れる |
| 専門用語だけで書く | 一文要約とポイントから作る |

## 品質基準

- 研究内容、意義、限界、公開前確認が揃っていること
- 広報文が専門外にも読めること
- PI、広報、知財、契約、共同機関の確認先が明示されていること
- 公開日時、問い合わせ先、謝辞・課題番号が確認対象になっていること

## Available Tools

- **Read**: 論文、成果台帳、研究計画、広報ガイドラインの読み込み
- **Write**: プレスリリース、Web文、取材メモ、確認表の作成
- **WebSearch** / **WebFetch**: 既存プレス、DOI、機関表記、FA表記の確認
- **Bash**: 文字数、表、版管理の整形
