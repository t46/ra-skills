What this project is

まず作ってみた、AI for RA のたたき台です。

ここでいう RA は Research Assistant ではなく、Research Administration、つまり研究を支える実務全体を指します。ただ、私はRA/URAの現場を十分に知っているわけではありません。だからこそ、まず形にして、実務に照らして直していきたいと考えています。

Problem

研究者も支援者も忙しすぎる

公募検索、申請、報告、予算、会議、広報、倫理、データ管理。研究の周辺には、重要だけれど時間を食う仕事が大量にあります。

Approach

業務を「スキル」にする

ただのプロンプト集ではなく、入力、手順、成果物、品質基準を持つ SKILL.md として整理します。

Goal

現場知で検証する

この手順で本当に使えるのか、責任分界や機関差に合っているのかを、現場の人に見てもらいながら直します。

Current draft skills

今あるスキルは、まず作ってみた試作です。

現在のリポジトリには、研究費、公募、申請書、報告書、予算、DMP、倫理、イベント運営などを支援するスキルがあります。ただし、完成品ではなく「これで現場に合っているか」を確認するための初期版です。

01

探す

`/funding-search` と `/policy-monitor` で、公募・政策・機会を見つける。

02

申請する

`/grant-proposal-review` と `/budget-planning` で、申請書と予算案を整える。

03

運営する

`/budget-execution` と `/progress-report` で、採択後の予算・報告を支える。

04

守る・広げる

`/ethics-review-support`、`/research-data-management`、`/event-organizer` で周辺実務を支援する。

Reality check

たぶん、RAの仕事はもっと広い。

MEXT / CRAMS の URA スキル標準 22 業務に照らすと、今の repo はまだ入口です。会議運営、予定調整、進捗管理、内部・対外折衝、契約、知財、安全管理、輸出管理、研究広報など、実務に詳しい方に確認したい領域が多く残っています。

Draft roadmap

次に試したいこと。

5月20日の勉強会では、完成品の発表というより、現場の人と一緒に「AI for RA の業務地図」を作るためのたたき台として共有します。使えそうなところ、ズレているところ、危ないところを教えてください。

Tier 0

会議・進捗・締切

`project-meeting-ops`、`project-progress-tracker`、`internal-deadline-tracker` などが必要そうだ、という仮説です。

Tier 1

専門部署へつなぐ

契約、知財、輸出管理、研究広報、安全管理は、判断代行ではなく論点整理に限るべきかもしれません。

Tier 2

研究戦略へ広げる

政策情報、研究力分析、公募情報をどう扱うべきかも、実務のニーズを聞きながら考えたいです。
