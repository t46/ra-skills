# 研究費 予算執行管理

## What This Does

採択後の研究費について、購入・発注・旅費・謝金・委託・年度末処理などの予算執行を支援する。公募要領、交付条件、所属機関規程に照らして、発注前に確認すべき論点と、証憑として残すべき情報を整理する。

- **対象ユーザー**: 研究者、URA、研究室秘書、部局事務、プロジェクトマネージャー
- **解決する課題**: 研究費で買えるか、どの費目か、いつまでに発注すべきか、証憑に何を残すべきか判断しづらい
- **期待される効果**: 不適切な支出、年度末の執行遅延、証憑不足、会計検査時の説明不能を減らす

## Required Inputs

- **資金種別**: 科研費、JST、AMED、NEDO、民間助成、共同研究費など
- **支出内容**: 購入したい物品・サービス、旅費、謝金、雇用、委託など
- **金額・時期**: 見積額、発注希望日、納品・実施予定日
- **研究計画との関係**: 何の研究目的・作業に使うか
- **所属機関の会計規程・運用**（任意）: 発注ルール、検収、旅費規程、謝金単価
- **現在の予算残額**（任意）: 費目別・年度別の残額

## What It Produces

1. **発注前チェック結果**: 支出可否、費目、確認先、保留事項
2. **証憑チェックリスト**: 見積書、発注書、納品書、検収記録、出張報告、謝金記録など
3. **執行スケジュール**: 内部締切、納品期限、検収期限、支払期限
4. **リスクコメント**: 会計検査・監査で説明を求められやすい点

## 手順

### 1. 支出の目的を確認する

まず「研究計画とどう関係する支出か」を1-2文で説明できる形にする。

```
この支出は、研究計画の [作業項目] において [目的] のために必要であり、
[成果物・実験・調査・解析] に直接使用する。
```

研究目的との関係を説明できない支出は、可否判断の前に用途を再整理する。

### 2. 資金ルールと機関ルールを分けて確認する

同じ支出でも、資金配分機関のルールと所属機関の会計ルールの両方を満たす必要がある。

| ルール | 確認内容 |
|--------|----------|
| 資金配分機関 | 使途制限、対象外経費、費目、年度、繰越、合算使用 |
| 所属機関 | 発注権限、相見積、検収、旅費規程、謝金単価、契約手続き |
| 研究計画 | 申請書・交付申請書の目的や予算内訳との整合性 |
| 契約・倫理 | 共同研究契約、倫理審査、個人情報、知財、輸出管理 |

### 3. 支出種別ごとに確認する

#### 3a. 物品購入

- 研究目的との関係
- 消耗品か設備備品か
- 単価・総額に応じた見積・契約手続き
- 納品時期と検収期限
- 汎用性が高い物品の場合、研究専用性を説明できるか
- 年度末購入の場合、年度内に納品・使用開始できるか

#### 3b. 旅費

- 出張目的が研究計画と対応しているか
- 学会参加、共同研究打合せ、調査、成果発表の区別
- 所属機関の旅費規程に沿った経路・日当・宿泊費か
- 出張命令、復命書、参加証、領収書などの証憑
- 私用日程が混在する場合の按分・説明

#### 3c. 人件費・謝金

- 業務内容、時間、単価、対象者が明確か
- 雇用契約、委嘱、謝金のどれに該当するか
- 研究代表者本人や親族など、支給できない相手でないか
- 勤務表、作業記録、成果物、支払記録が残るか
- 学生への支払いで教育・評価との利益相反がないか

#### 3d. 委託・外注

- 研究の本質的部分を丸投げしていないか
- 委託仕様書、見積、契約、成果物、検収方法が明確か
- 個人情報、秘密情報、知財の取り扱いが契約に含まれるか
- 再委託の可否、海外委託の可否を確認したか

### 4. 年度・締切を確認する

以下の期限を並べ、最も早い内部締切を基準にする。

- 発注依頼締切
- 契約締結期限
- 納品・役務完了期限
- 検収期限
- 支払書類提出期限
- 資金配分機関への報告期限
- 繰越・変更申請の期限

年度末の支出は「予算消化」に見えないよう、研究上の必要性と納品後の使用実態を説明できるようにする。

### 5. 証憑を整理する

支出前から、後で説明に必要な証憑を決めておく。

| 支出 | 残すもの |
|------|----------|
| 物品 | 見積書、発注書、納品書、検収記録、請求書、使用目的メモ |
| 旅費 | 出張申請、日程表、参加証、領収書、復命書、成果メモ |
| 謝金 | 依頼内容、単価根拠、作業記録、受領記録、成果物 |
| 委託 | 仕様書、契約書、成果物、検収記録、請求書 |

### 6. 判断結果を出力する

以下の形式で、研究者または事務担当者に渡せるメモを作る。

```markdown
## 発注前確認メモ

- 支出内容:
- 資金種別:
- 研究目的との関係:
- 想定費目:
- 支出可否: 可 / 要確認 / 不可
- 根拠:
- 追加確認先:
- 必要証憑:
- 期限:
- 注意点:
```

## 判断基準

- 「研究に関係がある」だけでなく、採択された研究計画との直接的な関係を説明できるか
- 資金配分機関のルールと所属機関の会計ルールの両方を満たしているか
- 年度内に納品・役務完了・検収できるか
- 証憑だけで第三者に支出目的を説明できるか
- 汎用性が高い物品、年度末支出、謝金、委託は重点的に確認する

## よくあるミスと対策

| ミス | 対策 |
|------|------|
| 「前にも買えたから今回も買える」と判断する | 資金種別、年度、研究計画、機関運用を毎回確認する |
| 発注後に費目や可否を相談する | 発注前確認メモを作ってから手続きを始める |
| 納品が年度を越える | 発注前に納期を確認し、難しい場合は繰越・変更申請を検討する |
| 旅費の研究目的が曖昧 | 出張目的、訪問先、成果、研究計画との関係を復命書に残す |
| 謝金の作業実態が説明できない | 作業日、時間、内容、成果物を記録する |

## 品質基準

- 支出可否、費目、確認先が区別されていること
- 判断根拠が公募要領、交付条件、所属機関規程、研究計画のどれに基づくか明記されていること
- 不明点を「要確認」として残し、確認先と期限を示していること
- 年度末・高額・汎用物品・委託・謝金のリスクが確認されていること
- 後から第三者が見ても支出目的を説明できる証憑リストになっていること

## Available Tools

- **Read**: 公募要領、交付条件、予算表、会計規程、見積書の読み込み
- **Write**: 発注前確認メモ、証憑チェックリスト、期限表の作成
- **WebSearch** / **WebFetch**: 資金配分機関・所属機関の最新ルール確認
- **Bash**: 予算残額計算、期限表の整形、CSV 集計
