# RAスキル試行テスト結果

## 目的

追加・改修したスキルが「それっぽい説明」ではなく、実務で使える成果物を出せるかを確認するため、架空だが現実的なRA業務ケースに適用した。

## テストケース

### ケースA: JST系大型公募の申請準備

入力:

- 公募: JST系の大型共同研究公募
- 外部締切: 2026-06-30 12:00
- 学内締切: 未確認
- 研究代表者: A教授
- 共同機関: 2機関
- 状況: 公募は見つけたが、申請書初稿、予算、共同機関承諾、e-Rad承認が未着手

使用スキル:

- `internal-deadline-tracker`
- `stakeholder-coordination`

期待する成果:

- 公式締切だけでなく、学内締切・共同機関締切・e-Rad承認を分ける
- 誰に何をいつ確認するかが分かる
- 申請可能性の危険度が分かる

### ケースB: 採択後3か月の月次進捗会議

入力:

- 研究期間: 2026-04-01から2029-03-31
- 現状: データ収集が1か月遅延、RA採用が未完了、予算執行は10%、倫理変更申請の要否が未確認
- 成果: 国内学会ポスター1件、プレプリント準備中
- 次の締切: 2026-07-15 月次会議、2026-09-30 中間進捗メモ

使用スキル:

- `project-progress-tracker`
- `project-meeting-ops`

期待する成果:

- 遅延と未決事項が分離される
- PI確認事項と専門部署確認が分かる
- 会議後のToDoが担当者・期限付きで残る

## 総合判定

| スキル | 判定 | 理由 |
|--------|------|------|
| `internal-deadline-tracker` | A- | 申請準備で最初に必要な逆算と未確認事項が出せる |
| `stakeholder-coordination` | A- | 関係者、未決事項、確認先の整理に実務性がある |
| `project-progress-tracker` | B+ | 月次ログとして有用。予算・成果台帳との接続をさらに強化したい |
| `project-meeting-ops` | A- | 会議をToDoとフォローアップに変換できる |

## 次の改善

- `progress-report` に、`project-progress-tracker` の月次ログを入力として使う手順を追加する。
- `funding-search` に、採用候補公募を `internal-deadline-tracker` へ渡す手順を追加する。
- `budget-execution` に、月次進捗と予算執行の乖離をチェックする手順を追加する。
- 契約・知財・輸出管理・研究広報の専門部署連携スキルを追加する。
