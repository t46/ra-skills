# 安全保障輸出管理インテーク

## What This Does

海外機関との共同研究、技術提供、試料・装置・ソフトウェア・データの海外送付、外国研究者・留学生受入、オンライン会議での技術説明などについて、輸出管理担当へ相談するための初期情報を整理する。該非判定や取引審査を代行せず、確認票と相談材料を作る。

- **対象ユーザー**: URA、研究支援担当者、PI、国際連携担当、部局事務
- **解決する課題**: 何が輸出管理確認の対象になり得るか分からず、相談が遅れる
- **期待される効果**: 技術提供・貨物輸出・相手先・用途の確認漏れを減らす

## Required Inputs

- **案件概要**: 共同研究、海外送付、技術説明、受入、国際会議など
- **相手先**: 機関名、国・地域、担当者、所属、最終需要者
- **提供内容**: 技術、データ、ソフト、試料、装置、図面、ノウハウ
- **用途**: 研究目的、利用場所、最終用途、再提供の有無
- **時期・方法**: 送付日、会議日、クラウド共有、メール、持ち出し、郵送
- **関連資料**（任意）: 研究計画、契約案、装置仕様、データ説明、招聘資料

## What It Produces

1. **輸出管理確認票ドラフト**: 相手先、用途、提供内容、時期、方法
2. **提供物リスト**: 貨物、技術、データ、ソフト、試料を分けた表
3. **相手先・用途確認リスト**: 国、機関、利用目的、再提供有無
4. **輸出管理担当への相談メール**: 背景、確認事項、資料を添えた依頼
5. **保留アクションリスト**: 確認完了まで止める作業

## 手順

### 1. 何を提供するのか分解する

| 種別 | 例 | 確認観点 |
|------|----|----------|
| 貨物 | 装置、試料、部品、薬品 | 海外送付、持ち出し |
| 技術 | 図面、実験条件、製造方法、ノウハウ | メール、会議、クラウド共有 |
| ソフトウェア | コード、モデル、解析ツール | ライセンス、提供方法 |
| データ | 実験データ、設計データ、個人情報 | 内容、公開範囲、契約 |
| 人の受入 | 留学生、外国研究者、訪問者 | アクセス範囲、研究内容 |

### 2. 相手先と用途を整理する

```markdown
| 相手先 | 国・地域 | 所属 | 役割 | 用途 | 最終需要者 | 再提供の有無 | 未確認事項 |
|--------|----------|------|------|------|------------|--------------|------------|
```

国名や所属だけで判断せず、輸出管理担当へ確認する。

### 3. 確認票ドラフトを作る

```markdown
## 輸出管理確認票ドラフト

- 案件名:
- 目的:
- 相手先:
- 提供内容:
- 提供方法:
- 提供時期:
- 用途:
- 関連契約:
- 添付資料:
- 確認したい事項:
```

### 4. 確認完了まで止める作業を明示する

```markdown
| 作業 | 状態 | 止める理由 | 再開条件 | 担当 |
|------|------|------------|----------|------|
```

例: 詳細図面の送付、未公開コードの共有、試料の海外郵送、クラウドフォルダ招待。

### 5. 輸出管理担当へ相談する

```markdown
件名: [輸出管理確認] {案件名} について

下記案件について、輸出管理上の確認をお願いいたします。

案件概要:
- {概要}

相手先・用途:
- {相手先}
- {用途}

提供予定:
- {提供物}
- {方法}
- {時期}

確認したい事項:
- 該非判定・取引審査の要否
- 確認完了まで保留すべき作業
```

## 判断基準

| 判断項目 | 見るポイント | 対応 |
|----------|--------------|------|
| 提供内容 | 技術・貨物・データ・ソフトのどれか | 種別ごとに整理 |
| 相手先 | 国、機関、担当者、最終需要者 | 未確認なら確認票へ |
| 用途 | 何に使われるか | 用途確認を記録 |
| 提供方法 | メール、会議、クラウド、郵送、持出し | 方法別に記録 |
| 開始前条件 | 確認完了前に進めてよいか | 保留作業にする |

## よくあるミスと対策

| ミス | 対策 |
|------|------|
| 物を送らなければ関係ないと思う | 技術提供やクラウド共有も確認対象に入れる |
| 国名だけで自己判断する | 相手先、用途、提供内容を輸出管理担当へ渡す |
| オンライン会議で詳細技術を説明する | 事前に提供内容を確認する |
| 留学生・訪問者のアクセス範囲を決めない | 受入前に研究内容とアクセス権限を整理する |

## 品質基準

- 提供内容、相手先、用途、方法、時期が整理されていること
- 該非判定・取引審査をRAが代行していないこと
- 確認完了まで保留する作業が明示されていること
- 契約、国際連携、安全、情報管理との接続があること

## Available Tools

- **Read**: 研究計画、契約案、装置仕様、データ説明、招聘資料の読み込み
- **Write**: 確認票、提供物リスト、相談メール、保留リストの作成
- **WebSearch** / **WebFetch**: 相手先公開情報、所属機関公開窓口の確認
- **Bash**: 提供物一覧、相手先表、期限表の整形
