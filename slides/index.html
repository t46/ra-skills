WHY02

研究のまわりには、AIに任せたい仕事が山ほどある。

研究費公募検索、要件確認、申請書、予算、e-Rad。
採択後会議、進捗、予算執行、報告書、評価対応。
周辺実務倫理、DMP、契約、知財、広報、イベント。
AI for Science の前に、AI for Research Administration
AI for Science というと、研究の本体をAIがやる話になりがちです。でも実際には、研究の周辺にある仕事がとても多い。公募を探す、申請書を整える、予算を組む、採択後に会議を回す、報告書を書く。ここをAIが支えられるだけでも、研究現場にはかなり効くはずです。
WHAT IS A SKILL?03
SKILL

プロンプト集ではなく、業務手順のパッケージ。

入力、手順、成果物、品質基準を `SKILL.md` として書く。AI エージェントはそれを読んで、具体的な作業に入れる。

Input → Procedure → Output → Quality Gate
ポイントは、単なるプロンプト集ではないということです。スキルには、何を入力として受け取るか、どの順番で確認するか、最終的に何を出すか、品質基準は何かを書きます。つまり現場のノウハウを、AIが実行しやすい業務手順としてパッケージ化するイメージです。

03内部/対外折衝

今の repo は「書類作成支援」に寄っている。次は、研究プロジェクトを実際に回すスキルが必要。

MEXT / CRAMS の URA 22 業務と照合
その後、MEXTやCRAMSのURAスキル標準に照らして監査しました。すると、今のリポジトリは公募、申請、予算、報告の入口にはなっている一方で、実際のRA業務のかなり重要な部分が薄い。内部調整、対外折衝、定例会議、進捗管理、契約、知財、研究広報などです。
ROADMAP07

次に作るべきスキル。

Tier 1collaboration-contract-intake / ip-disclosure-triage / export-control-intake / research-pr-support
Tier 2research-strategy-brief / international-collaboration-logistics / project-closeout
書類から、運営・調整・専門部署連携へ
次に作るべきものはかなり明確です。まずは会議、進捗、締切、関係者調整。ここは多くの研究プロジェクトに共通していて、AIがすぐ役に立ちやすい。次に契約、知財、輸出管理、広報。ただしここはAIが判断を代行するのではなく、専門部署に渡すための論点整理をする形がよいと思っています。
INVITATION08
