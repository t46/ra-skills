# 研究戦略ブリーフ作成

## What This Does

政策動向、研究力分析、公募情報、学内資源、競合・連携候補を統合し、部局・研究グループ・センター向けの短い研究戦略メモを作る。単なる情報収集ではなく、「どの機会に、誰が、何を準備すべきか」まで落とす。

- **対象ユーザー**: URA、研究戦略担当、部局執行部、センター長、PIグループ
- **解決する課題**: 政策情報、公募情報、研究力分析が別々に存在し、具体的な行動に変換されない
- **期待される効果**: 重点領域、候補PI、チーム形成、次期公募準備、学内施策を議論しやすくする

## Required Inputs

- **対象範囲**: 部局、研究グループ、センター、テーマ、期間
- **政策・公募情報**: 政府方針、FA公募、審議会資料、学内重点施策
- **研究力情報**: 論文、被引用、KAKEN、researchmap、受賞、外部資金、共同研究
- **学内資源**: 研究者、設備、URA、事務、産学連携、国際連携、既存プロジェクト
- **目的**: 部局会議、戦略会議、公募準備、拠点形成、予算要求など
- **制約**（任意）: 締切、候補者、重点分野、公開可否

## What It Produces

1. **研究戦略ブリーフ**: 背景、機会、強み、ギャップ、推奨アクション
2. **機会マップ**: 政策・公募・研究力・学内資源の対応表
3. **候補テーマ・候補チーム案**: PI、分担、共同機関、準備ToDo
4. **意思決定メモ**: 部局・研究グループで決めるべき事項
5. **次アクション表**: 担当者、期限、確認先、必要資料

## 手順

### 1. ブリーフの用途を定義する

```markdown
用途:
- 読者:
- 決めたいこと:
- 使う会議:
- 期限:
- 公開範囲:
```

読者が部局執行部なのか、候補PIなのかで粒度を変える。

### 2. 情報源を分ける

| 情報 | 例 | 注意 |
|------|----|------|
| 政策 | MEXT、CSTI、審議会、FA戦略 | 最新公式資料を確認 |
| 公募 | JST、JSPS、AMED、NEDO、民間助成 | 締切・応募資格・推薦枠 |
| 研究力 | 論文、KAKEN、OpenAlex、researchmap | 指標の限界を明記 |
| 学内資源 | 設備、研究者、既存拠点、支援体制 | 非公開情報の扱いに注意 |
| 外部連携 | 企業、自治体、海外機関 | 契約・知財・国際確認 |

### 3. 機会マップを作る

```markdown
| 機会/政策 | 関連公募 | 学内の強み | 足りない要素 | 候補者/部署 | 次アクション |
|-----------|----------|------------|--------------|-------------|--------------|
```

### 4. 戦略ブリーフを書く

```markdown
# 研究戦略ブリーフ

## 要旨

## 背景

## いま見えている機会

## 学内の強み

## ギャップ・リスク

## 推奨アクション

## 次に決めること
```

推奨アクションは、情報提供、チーム形成、ヒアリング、公募準備、学内施策、外部連携に分ける。

### 5. 次アクションに落とす

```markdown
| アクション | 目的 | 担当 | 期限 | 確認先 | 成果物 |
|------------|------|------|------|--------|--------|
```

## 判断基準

| 判断項目 | 見るポイント | 対応 |
|----------|--------------|------|
| 根拠 | 政策・公募・指標に出典があるか | URLと確認日を記録 |
| 実行可能性 | 候補者・期限・支援体制があるか | ToDo化する |
| 指標の限界 | 分野差・DB差・同姓同名があるか | 限界を明記 |
| 機関戦略 | 学内方針と整合するか | 関係部署に確認 |
| 公開範囲 | 非公開情報が含まれるか | 共有範囲を明示 |

## よくあるミスと対策

| ミス | 対策 |
|------|------|
| 政策情報の要約で終わる | 候補テーマ・チーム・ToDoまで落とす |
| 指標をランキングのように扱う | 分野差と限界を書く |
| 公募締切から逆算しない | `internal-deadline-tracker` へ渡す |
| 誰が読むか不明なメモになる | 読者と会議用途を最初に置く |

## 品質基準

- 政策、公募、研究力、学内資源が一枚の判断材料になっていること
- 推奨アクションに担当者、期限、成果物があること
- 指標と推測を区別していること
- `policy-monitor`, `research-metrics`, `funding-search` の出力を統合できること

## Available Tools

- **Read**: 政策資料、公募要領、研究力データ、学内資料の読み込み
- **Write**: 戦略ブリーフ、機会マップ、候補チーム表、ToDo表の作成
- **WebSearch** / **WebFetch**: 政府・FA・大学・研究者情報の確認
- **Bash**: 表、CSV、集計、出典一覧の整形
