What this project is

リポジトリを見る前に、何のためのものか分かるように。

ここでいう RA は Research Assistant ではなく、Research Administration、つまり研究を支える実務全体を指します。URA、大学事務、研究者、研究所スタッフが担っている仕事を、AI に渡せる単位へ分解します。

Problem

研究者も支援者も忙しすぎる

公募検索、申請、報告、予算、会議、広報、倫理、データ管理。研究の周辺には、重要だけれど時間を食う仕事が大量にあります。

Approach

業務を「スキル」にする

ただのプロンプト集ではなく、入力、手順、成果物、品質基準を持つ SKILL.md として整理します。

Goal

現場知を共有できる形にする

属人的な RA / URA ノウハウを、他の人と AI エージェントが再利用できる形に変換します。

Current skills

今あるスキルで、まず何ができるか。

現在のリポジトリには、研究費、公募、申請書、報告書、予算、DMP、倫理、イベント運営などを支援するスキルがあります。

01

探す

`/funding-search` と `/policy-monitor` で、公募・政策・機会を見つける。

02

申請する

`/grant-proposal-review` と `/budget-planning` で、申請書と予算案を整える。

03

運営する

`/budget-execution` と `/progress-report` で、採択後の予算・報告を支える。

04

守る・広げる

`/ethics-review-support`、`/research-data-management`、`/event-organizer` で周辺実務を支援する。

Reality check

ただし、RAの仕事はもっと広い。

MEXT / CRAMS の URA スキル標準 22 業務に照らすと、今の repo はまだ入口です。会議運営、予定調整、進捗管理、内部・対外折衝、契約、知財、安全管理、輸出管理、研究広報が今後の重点です。

Next roadmap

次に作るべきもの。

5月20日の勉強会では、完成品の発表というより、現場の人と一緒に「AI for RA の業務地図」を作るためのたたき台として共有します。

Tier 0

会議・進捗・締切

`project-meeting-ops`、`project-progress-tracker`、`internal-deadline-tracker` を最優先で追加する。

Tier 1

専門部署へつなぐ

契約、知財、輸出管理、研究広報、安全管理は、判断代行ではなく論点整理スキルとして作る。

Tier 2

研究戦略へ広げる

政策情報、研究力分析、公募情報を統合し、部局や研究組織の意思決定に使える形へ広げる。
