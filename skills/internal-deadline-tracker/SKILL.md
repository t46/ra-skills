# 研究費申請 内部締切トラッカー

## What This Does

研究費公募の公式締切から逆算し、学内締切、部局締切、推薦・承認、e-Rad入力、申請書レビュー、予算確認、押印・機関承認などの内部タスクを整理する。公募を見つけるだけでなく、研究者と事務が締切に間に合う実行計画を作る。

- **対象ユーザー**: URA、研究支援担当者、部局事務、PI、研究室秘書
- **解決する課題**: 外部締切だけ見て動き始め、学内承認やe-Rad承認に間に合わない
- **期待される効果**: 申請漏れ、差し戻し、内部承認遅延、資料未完成を減らす

## Required Inputs

- **公募情報**: 公募名、FA、公式URL、外部締切、提出方法
- **申請者情報**: PI、所属、共同研究者、研究機関数
- **提出物**: 申請書、予算書、承諾書、推薦書、倫理/COI関連書類など
- **機関内ルール**（任意）: 学内締切、部局締切、承認者、事務担当、推薦枠
- **レビュー希望**（任意）: 申請書レビュー、予算確認、模擬面接など
- **現在日**: 逆算の基準日

## What It Produces

1. **逆算スケジュール表**: 外部締切、学内締切、部局締切、作業期限
2. **担当者別ToDo**: PI、URA、部局事務、共同機関ごとの作業
3. **不足情報リスト**: 公式要領・機関通知・承認者・様式などの不足
4. **リマインド文面**: PIや共同研究者へ送る依頼メール
5. **危険度判定**: 余裕あり / 要注意 / 高リスク / 申請困難

## 手順

### 1. 公式情報を確認する

必ず一次情報を確認し、確認日を記録する。

優先順位:

1. FA公式の公募要領・様式・Q&A
2. e-Radまたは申請システム上の公募情報
3. 所属機関の学内通知・部局通知
4. 二次情報・まとめサイト

```markdown
確認した根拠:
- FA公式URL:
- 公募要領版/公開日:
- 学内通知URL/ファイル:
- 確認日:
```

### 2. 締切の種類を分ける

外部締切だけでなく、研究者が実際に守るべき内部締切を並べる。

| 締切種別 | 例 | 確認先 |
|----------|----|--------|
| 外部締切 | FAへの提出期限 | 公募要領、e-Rad |
| 機関承認締切 | 研究機関による承認期限 | 研究推進課、URA |
| 学内提出締切 | PIから事務への提出期限 | 学内通知 |
| 部局締切 | 部局内レビュー・推薦締切 | 部局事務 |
| 共同機関締切 | 分担者承諾、予算確認 | 共同機関担当 |
| レビュー締切 | 申請書レビュー依頼期限 | URA、外部レビュー |

### 3. 逆算表を作る

締切から逆算し、作業に必要なバッファを入れる。バッファは機関運用で変わるため、根拠がなければ「仮置き」と明記する。

```markdown
| 日付 | 締切/作業 | 担当 | 根拠 | 状態 | 備考 |
|------|-----------|------|------|------|------|
```

逆算で入れる典型タスク:

- 応募資格・重複制限の確認
- PIへの応募意思確認
- 共同研究者・共同機関の確定
- 予算案作成と機関ルール確認
- 申請書初稿
- URAレビュー
- 部局レビュー・推薦
- e-Rad入力
- 機関承認
- 最終PDF確認

### 4. 危険度を判定する

| 判定 | 目安 | 対応 |
|------|------|------|
| 余裕あり | レビューと差し戻しの時間がある | 通常スケジュール |
| 要注意 | 主要資料の初稿期限が近い | レビュー範囲を絞る |
| 高リスク | 学内締切・承認に間に合わない可能性 | 事務に即確認、申請範囲を縮小 |
| 申請困難 | 公式・学内締切を過ぎている | 次回公募、別資金を検討 |

断定できない場合は、判断ではなく「確認依頼」を出す。

### 5. 担当者別ToDoを作る

```markdown
## 担当者別ToDo

### PI
- {タスク}（期限: {日付}、確認先: {担当}）

### URA
- {タスク}（期限: {日付}）

### 部局事務
- {タスク}（期限: {日付}）

### 共同研究者/共同機関
- {タスク}（期限: {日付}）
```

### 6. リマインドを作る

```markdown
件名: [依頼] {公募名} 申請準備の締切確認

{公募名} について、外部締切は {外部締切} ですが、
学内手続きのため {学内締切} までに下記をご準備ください。

お願いしたいこと:
- {依頼1}
- {依頼2}

未確認事項:
- {確認事項}: {確認先} に確認中
```

## 判断基準

| 判断項目 | 見るポイント | 対応 |
|----------|--------------|------|
| 根拠 | 公式要領・学内通知を確認したか | 確認日とURLを残す |
| 内部締切 | 外部締切より前の承認期限があるか | 学内担当に確認 |
| 推薦枠 | 機関内推薦・件数制限があるか | 部局/本部に確認 |
| 共同機関 | 分担者承諾・予算確認が必要か | 共同機関締切を置く |
| e-Rad | 研究者情報・機関承認が必要か | 早めに事務確認 |

## よくあるミスと対策

| ミス | 対策 |
|------|------|
| FAの締切だけを見て予定を立てる | 学内締切と機関承認期限を先に確認する |
| 公募要領の古い版を使う | 版、公開日、確認日を記録する |
| 推薦枠・重複制限を見落とす | 応募資格と件数制限を初日に確認する |
| e-Rad入力を最後に回す | 研究者情報、分担者、機関承認を先に確認する |
| 共同機関の承諾が遅れる | 共同機関用の内部締切を別に置く |

## 品質基準

- 外部締切、学内締切、部局締切、機関承認期限が分かれていること
- 公式情報のURL、版、確認日が記録されていること
- 担当者別ToDoに期限と確認先があること
- 推薦枠、重複制限、応募資格、e-Rad承認の確認が含まれていること
- 危険度判定があり、リスク時の次アクションが明確であること

## Available Tools

- **Read**: 公募要領、学内通知、様式、過去スケジュールの読み込み
- **Write**: 逆算表、ToDo、リマインド文面の作成
- **WebSearch** / **WebFetch**: FA公式、公募要領、e-Rad情報、学内公開通知の確認
- **Bash**: 日付計算、締切表の整形、CSV作成
