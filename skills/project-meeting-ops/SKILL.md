# 研究プロジェクト会議運営

## What This Does

研究プロジェクトの定例会議、キックオフ、進捗確認会、評価前打合せを、決定事項と次アクションが残る形で運営する。単なる議事録作成ではなく、会議前の目的整理、日程調整、資料依頼、当日の論点管理、会議後の未決事項フォローまで支援する。

- **対象ユーザー**: URA、研究支援担当者、プロジェクトマネージャー、PI、部局事務
- **解決する課題**: 会議が情報共有だけで終わり、誰がいつ何をするかが曖昧になる
- **期待される効果**: 進捗遅延、確認漏れ、資料未提出、次回会議までの放置を減らす

## Required Inputs

- **会議の目的**: キックオフ、月次進捗、予算確認、評価準備、共同機関連絡など
- **参加者**: 氏名、所属、役割、必須/任意参加の区別
- **候補日時・制約**: 候補日、締切、タイムゾーン、参加不可時間
- **決めたい事項**: 会議で決定すべき項目
- **共有したい情報**: 進捗、予算、成果、リスク、依頼事項
- **前回ToDo**（任意）: 担当者、期限、完了状況
- **会議資料**（任意）: 申請書、採択通知、予算表、進捗表、評価資料など

## What It Produces

1. **日程調整文面**: 候補日時、回答期限、参加必須者を含む依頼文
2. **アジェンダ**: 目的、論点、所要時間、担当者、決定事項の欄
3. **議事録テンプレート/議事録**: 決定事項、未決事項、ToDo、次回確認事項を分離
4. **フォローアップ文面**: 会議後に参加者へ送る確認メール
5. **欠席者共有メモ**: 欠席者が最低限追える要約

## 手順

### 1. 会議を開く必要性を確認する

まず会議の目的を1文で書く。目的が「情報共有」だけの場合は、メール・共有ドキュメントで足りるかを確認する。

```markdown
今回の会議目的:
- 決めること:
- 相談すること:
- 共有だけでよいこと:
```

会議が必要な典型例:

- 研究計画、予算、契約、倫理、安全、知財に関する判断が必要
- 複数部署・共同機関の合意が必要
- 評価、報告、申請の締切から逆算して進捗確認が必要
- リスクや遅延があり、対応方針を決める必要がある

### 2. 参加者と役割を分ける

参加者を役割で整理し、必須参加者と情報共有のみの人を分ける。

| 役割 | 典型例 | 会議で期待すること |
|------|--------|--------------------|
| 意思決定者 | PI、研究代表者、部局責任者 | 方針決定、優先順位判断 |
| 実務担当 | URA、事務、PM、秘書 | 手続き、期限、資料準備 |
| 専門部署 | 経理、契約、知財、倫理、安全、情報基盤 | 制度・規程の確認 |
| 共同機関 | 共同研究者、企業、委託先 | 分担、成果物、契約条件 |

### 3. 日程調整を行う

日程調整では、単に空き時間を聞くのではなく、締切から逆算した回答期限を置く。

```markdown
件名: [日程調整] {プロジェクト名} {会議目的}

{会議目的} のため、下記候補で打合せを設定したく存じます。
恐れ入りますが、{回答期限} までに参加可否をご返信ください。

候補:
- {候補1}
- {候補2}
- {候補3}

主な論点:
- {論点1}
- {論点2}

事前に確認いただきたい資料:
- {資料名}
```

### 4. アジェンダを作る

アジェンダは「報告」ではなく「決定・確認・相談」に分ける。

| 時間 | 論点 | 種別 | 担当 | 期待する出力 |
|------|------|------|------|--------------|
| 5分 | 前回ToDo確認 | 確認 | URA | 完了/未完了の整理 |
| 15分 | 進捗と遅延 | 相談 | PI/担当者 | 対応方針 |
| 10分 | 予算執行 | 決定 | PI/経理 | 発注可否・確認先 |
| 10分 | 次回までの作業 | 決定 | 全員 | 担当者と期限 |

### 5. 議事録を作成する

議事録は発言録ではなく、後で動ける管理表にする。

```markdown
# 議事録

- 会議名:
- 日時:
- 参加者:
- 欠席者:
- 作成日:

## 決定事項
| No | 決定事項 | 根拠/背景 | 影響範囲 |
|----|----------|-----------|----------|

## ToDo
| No | タスク | 担当 | 期限 | 確認先 | 状態 |
|----|--------|------|------|--------|------|

## 未決事項
| No | 未決事項 | 何が不足しているか | 確認先 | 期限 |
|----|----------|--------------------|--------|------|

## 次回会議
- 目的:
- 候補時期:
- 次回までに必要な資料:
```

### 6. 会議後フォローを送る

会議後24時間以内を目安に、決定事項とToDoだけを短く送る。

```markdown
件名: [共有] {会議名} 議事メモとToDo

本日の打合せの決定事項とToDoを共有します。
認識違いがあれば {確認期限} までにご連絡ください。

決定事項:
- {決定事項}

ToDo:
- {担当者}: {タスク}（期限: {期限}）

未決事項:
- {未決事項}: {確認先} に確認
```

## 判断基準

| 判断項目 | 見るポイント | 対応 |
|----------|--------------|------|
| 会議の必要性 | その場で決めることがあるか | なければメール共有にする |
| 参加者 | 決定者・実務者・専門部署が揃うか | 必須参加者を明示する |
| 記録 | 決定事項、ToDo、未決事項が分かれているか | 議事録テンプレートで分離する |
| 期限 | 外部締切・内部締切と整合しているか | ToDoに期限を入れる |

## よくあるミスと対策

| ミス | 対策 |
|------|------|
| アジェンダが「進捗共有」だけになる | 決めたい事項と相談したい事項を先に書く |
| 議事録が発言録になり、ToDoが残らない | 決定事項・ToDo・未決事項の表にする |
| 欠席者に文脈が伝わらない | 欠席者向けに結論と依頼だけのメモを作る |
| 専門部署の確認が後回しになる | 会議中に確認先と期限をToDo化する |
| 次回会議まで誰も動かない | 会議後24時間以内にフォローアップを送る |

## 品質基準

- 会議目的、決めること、相談することが区別されていること
- 決定事項、ToDo、未決事項、確認先、期限が残っていること
- 欠席者が読んでも最低限の判断と依頼事項を理解できること
- 専門部署が必要な論点をRAが単独判断していないこと
- 次回会議の目的と準備物が明示されていること

## Available Tools

- **Read**: 進捗表、前回議事録、予算表、申請書、採択通知の読み込み
- **Write**: 日程調整文、アジェンダ、議事録、フォローアップ文の作成
- **WebSearch** / **WebFetch**: 公募・FA・機関の最新締切や会議要件の確認
- **Bash**: ToDo表、参加者表、期限表の整形
