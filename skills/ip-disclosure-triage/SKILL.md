# 知財相談トリアージ

## What This Does

研究成果に発明、ソフトウェア、データベース、ノウハウ、著作物、商標、ライセンス可能性が含まれる場合に、知財担当へ相談するための初期情報を整理する。特許性や権利帰属を判断するスキルではなく、公開前に確認すべき材料を揃える。

- **対象ユーザー**: URA、産学連携担当、PI、研究支援担当者、学生・若手研究者
- **解決する課題**: 論文投稿、学会発表、プレス公開の前に知財確認が漏れる
- **期待される効果**: 発明届の遅れ、共同出願論点の見落とし、公開による権利化機会喪失を減らす

## Required Inputs

- **成果概要**: 何を発明・作成・発見したか、従来との差分
- **公開予定**: 論文投稿、学会発表、プレプリント、プレス、GitHub公開、データ公開の日付
- **関係者**: 発明者候補、共著者、共同研究者、学生、企業、海外機関
- **資金・契約**: 研究費、共同研究契約、受託、NDA、データ提供契約
- **資料**（任意）: 論文案、スライド、図、コード、データ、契約案、過去発明届

## What It Produces

1. **知財相談メモ**: 成果概要、新規性候補、公開予定、関係者、資金・契約
2. **公開前確認リスト**: 論文、学会、プレス、ソフトウェア、データの公開可否確認
3. **共同出願・権利関係の論点表**: 企業・共同機関・学生・外部資金との関係
4. **知財担当への相談メール**: 資料と確認事項を添えた依頼文
5. **次アクション表**: 発明届、追加資料、公開延期相談など

## 手順

### 1. 公開予定を先に確認する

知財確認では、公開日が最重要の制約になる。

```markdown
公開予定:
- 論文投稿:
- 学会発表:
- プレプリント:
- プレスリリース:
- GitHub / データ公開:
- その他公開:
```

公開予定が近い場合は、緊急度を上げて知財担当へ相談する。

### 2. 成果の種類を分類する

| 種類 | 例 | 確認先 |
|------|----|--------|
| 発明 | 装置、方法、材料、アルゴリズム、用途 | 知財担当 |
| ソフトウェア | 研究コード、Webツール、パッケージ | 知財・情報基盤 |
| データベース | 整備データ、注釈データ、ベンチマーク | 知財・RDM |
| ノウハウ | 実験条件、製造条件、運用手順 | 知財・契約 |
| 著作物 | 図表、教材、文章、画像 | 知財・広報 |

### 3. 知財相談メモを作る

```markdown
## 知財相談メモ

- 成果名:
- 成果概要:
- 従来との差分:
- 想定用途:
- 関係者:
- 関連資金:
- 共同研究・契約:
- 公開予定:
- 添付資料:
- 確認したい事項:
```

### 4. 関係者と契約を確認する

```markdown
| 関係者/機関 | 関与内容 | 契約・資金 | 権利関係の未決事項 | 確認先 |
|-------------|----------|------------|----------------------|--------|
```

学生、企業、共同機関、海外機関が関与する場合は、所属機関の手続きに従う。

### 5. 知財担当へ相談する

```markdown
件名: [知財相談] {成果名} の公開前確認について

下記成果について、論文/発表/公開前に知財上の確認をお願いいたします。

公開予定:
- {日付・公開種別}

成果概要:
- {概要}

関係者・契約:
- {関係者}

確認したい事項:
- 発明届の要否
- 公開前に止めるべき事項
- 共同機関・企業との確認事項
```

## 判断基準

| 判断項目 | 見るポイント | 対応 |
|----------|--------------|------|
| 公開時期 | いつ公開されるか | 公開前に知財確認 |
| 発明可能性 | 技術的特徴・用途・方法があるか | 発明届要否を相談 |
| 共同関係 | 企業・共同機関・学生が関与するか | 契約・権利関係を確認 |
| 資金 | 外部資金の成果扱いがあるか | 研究費ルール確認 |
| ソフト・データ | ライセンスや公開条件があるか | RDM・情報基盤と連携 |

## よくあるミスと対策

| ミス | 対策 |
|------|------|
| 論文受理後に初めて知財相談する | 投稿・発表前に相談する |
| 発明者と共著者を同一視する | 知財担当へ発明者候補を相談する |
| GitHub公開を軽く扱う | ソフトウェア公開も公開前確認に入れる |
| 企業共同研究の契約を見ない | `collaboration-contract-intake` と連携する |

## 品質基準

- 公開予定、成果概要、関係者、契約、資金が整理されていること
- 特許性・権利帰属をRAが断定していないこと
- 知財担当が初回相談で判断に入れる材料が揃っていること
- 広報、論文投稿、データ公開との接続が明示されていること

## Available Tools

- **Read**: 論文案、発表資料、契約案、研究計画、成果台帳の読み込み
- **Write**: 知財相談メモ、確認リスト、相談メールの作成
- **WebSearch** / **WebFetch**: 公開予定、DOI、先行公開情報の確認
- **Bash**: 期限表、関係者表、資料一覧の整形
