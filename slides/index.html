Act I · Why
02 / 11
研究の周辺にある仕事

研究のまわりには、AIに任せたい仕事が山ほどある。

01
研究費
公募検索、要件確認、申請書、予算、e-Rad。締切は学内の手続き込みで動く。
02
採択後
会議、進捗、予算執行、報告書、評価対応。研究は採択後の運営で止まりやすい。
03
周辺実務
倫理、DMP、契約、知財、広報、イベント。専門部署につなぐ前処理も大きい。

Act II · Definition
03 / 11
What is a skill?

プロンプト集ではなく、業務手順のパッケージ。

入力、手順、成果物、品質基準を SKILL.md として書く。AI エージェントはそれを読んで、具体的な作業に入れる。

Input → Procedure → Output → Quality Gate
Skill = operational knowledge package

05 / 11
個別スキルの例

たとえば、こんな仕事を頼めます。

funding-search
出せそうな公募を探す
公募一覧、適合度、応募資格、公式URL、締切、確認事項。
internal-deadline-tracker
締切を逆算する
外部締切、学内締切、部局承認、e-Rad承認、担当者別ToDo。
grant-proposal-review
申請書をレビューする
審査観点ごとの弱点、改善案、確認すべき根拠。
project-meeting-ops
会議を回す
日程調整文、アジェンダ、議事録、会議後ToDo。
project-progress-tracker
進捗とリスクを追う
月次サマリー、マイルストーン、リスク表、成果登録表。
stakeholder-coordination
関係者を整理する
関係者マップ、合意事項、未決事項、確認依頼メール。
ポイントは、成果物が「次に動ける形」になっていること

New Layer · Operations
09 / 11
今回追加したもの

「採択後を回す」ための、運営系スキルを足しました。

01
締切
外部締切から、学内締切、部局承認、e-Rad承認、共同機関の締切を逆算する。
02
会議
日程調整、アジェンダ、議事録、欠席者共有、会議後ToDoまで作る。
03
進捗
月次サマリー、マイルストーン、リスク、成果登録表を報告書につなぐ。
04
調整
関係者、合意事項、未決事項、専門部署への確認依頼を整理する。
試行テストでは、JST系公募準備と採択後月次会議に当ててみた
