# 研究プロジェクト終了処理

## What This Does

研究プロジェクト終了時に、最終報告、予算執行締切、契約終了、成果台帳、データ保存、知財、広報、機器・雇用・共同機関、次期資金獲得への接続を整理する。終了直前に慌てて報告書を書くのではなく、終わり方を管理表にする。

- **対象ユーザー**: URA、研究支援担当者、PI、プロジェクトマネージャー、部局事務
- **解決する課題**: 最終報告、予算、成果、データ、契約が別々に進み、終了後に漏れが出る
- **期待される効果**: 報告漏れ、支出期限超過、成果公開漏れ、データ保存漏れを減らす

## Required Inputs

- **プロジェクト情報**: 制度名、課題名、研究期間、終了日、報告期限
- **予算情報**: 残額、支出締切、発注・納品・検収期限、繰越可否
- **成果情報**: 成果台帳、論文、発表、特許、データ、広報、受賞
- **契約・共同機関**: 契約終了条件、成果物、請求、報告、データ・試料返却
- **データ・資産**: DMP、保存場所、公開範囲、機器、アカウント、雇用
- **次期展開**（任意）: 後継公募、継続計画、広報予定

## What It Produces

1. **終了処理チェックリスト**: 報告、予算、契約、成果、データ、知財、広報、雇用の確認表
2. **最終報告準備表**: 必要資料、担当、期限、未回収情報
3. **予算終了ToDo表**: 発注、納品、検収、旅費、謝金、残額確認
4. **成果・データ公開計画**: 台帳、リポジトリ、知財・広報確認
5. **次期資金ブリッジメモ**: 成果、未解決課題、次に狙う公募

## 手順

### 1. 終了期限を分解する

外部の終了日だけでなく、学内の支出・検収・報告締切を確認する。

```markdown
期限:
- 研究期間終了日:
- 最終報告提出:
- 支出締切:
- 発注締切:
- 納品・検収締切:
- 共同機関報告:
- データ公開/保存:
```

### 2. 終了処理チェックリストを作る

| 領域 | 確認事項 | 確認先 |
|------|----------|--------|
| 報告 | 最終報告、成果一覧、評価資料 | FA、研究推進課 |
| 予算 | 残額、締切、証憑、検収 | 経理 |
| 契約 | 契約終了、成果物、請求、秘密保持 | 契約担当 |
| 成果 | 論文、特許、データ、広報、受賞 | PI、知財、広報 |
| データ | 保存、公開、アクセス権限、DMP更新 | RDM、情報基盤 |
| 人・資産 | 雇用終了、機器、アカウント、貸与物 | 部局事務 |
| 次期展開 | 後継公募、共同研究、広報 | URA、PI |

### 3. 最終報告へ渡す材料を集める

`research-output-registry` と `project-progress-tracker` の出力を使い、未回収情報を抽出する。

```markdown
| 資料 | 用途 | 担当 | 期限 | 状態 | 未決事項 |
|------|------|------|------|------|----------|
```

### 4. 予算終了ToDoを作る

```markdown
| 支出項目 | 残作業 | 発注期限 | 納品/検収期限 | 担当 | 確認先 | 状態 |
|----------|--------|----------|---------------|------|--------|------|
```

制度・機関差が大きいため、費目や繰越可否は必ず公式資料・経理で確認する。

### 5. 次期資金ブリッジメモを作る

```markdown
## 次期展開メモ

- 主な成果:
- まだ残る課題:
- 次に必要な資金:
- 候補公募:
- 共同機関:
- 申請に使える実績:
```

## 判断基準

| 判断項目 | 見るポイント | 対応 |
|----------|--------------|------|
| 期限 | FA締切と学内締切が分かれているか | 逆算表にする |
| 予算 | 支出・納品・検収の期限が合うか | 経理確認 |
| 成果 | 報告対象が台帳化されているか | 未回収情報をPIへ確認 |
| データ | 保存・公開・アクセス権限が決まっているか | RDM担当へ確認 |
| 契約 | 終了条件・成果物・秘密保持が残るか | 契約担当へ確認 |

## よくあるミスと対策

| ミス | 対策 |
|------|------|
| 最終報告だけを終了処理だと思う | 予算、契約、成果、データ、雇用も見る |
| 支出締切と納品検収締切を混同する | 経理確認表で分ける |
| 成果台帳がなく報告直前に探す | `research-output-registry` を使う |
| データ保存とアカウント整理を忘れる | DMPと情報基盤確認を入れる |

## 品質基準

- 終了日、報告、支出、検収、契約、データの期限が分かれていること
- 担当者、確認先、未決事項、状態があること
- 最終報告だけでなく、次期資金・成果展開へつながること
- 専門部署判断をRAが代行していないこと

## Available Tools

- **Read**: 報告書、予算表、契約書、成果台帳、DMP、議事録の読み込み
- **Write**: 終了処理表、最終報告準備表、予算ToDo、次期展開メモの作成
- **WebSearch** / **WebFetch**: FA公式報告要領、次期公募情報の確認
- **Bash**: 期限表、成果一覧、残作業表の整形
