# スキルの日本語名

## What This Does

このスキルが支援する RA / URA 業務を説明する。

- **対象ユーザー**: 研究者、URA、研究支援担当者、部局事務など
- **解決する課題**: 何に困っている人を助けるのか
- **期待される効果**: どの成果物や判断が改善されるのか

## Required Inputs

- **必須入力1**: 説明
- **必須入力2**: 説明
- **任意入力**（任意）: 説明

## What It Produces

1. **成果物1**: 形式、用途、含める項目
2. **成果物2**: 形式、用途、含める項目
3. **確認リスト**: 判断漏れや差し戻しを防ぐチェック項目

## 手順

### 1. 入力情報を確認する

ユーザーから受け取った情報を整理し、不足情報を確認する。

### 2. 適用される制度・ルールを確認する

公募要領、所属機関の規程、公式マニュアルなど、判断に使う一次情報を確認する。

### 3. 作業を実行する

業務の中心となる作業をステップバイステップで行う。

### 4. 品質チェックを行う

成果物が利用目的に耐えるかを確認し、必要に応じて修正する。

## 判断基準

| 判断項目 | 見るポイント | 対応 |
|----------|--------------|------|
| ルール確認 | 公募要領・機関規程に根拠があるか | 根拠を明記する |
| 不確実性 | 機関差・年度差・制度変更の可能性があるか | 確認先を明記する |
| 期限 | 内部締切・外部締切の両方を見ているか | 余裕を持った日程にする |

## よくあるミスと対策

| ミス | 対策 |
|------|------|
| 所属機関ごとの運用差を見落とす | 汎用ルールと機関固有ルールを分けて記述する |
| 古い制度情報を使う | 公式情報の確認日を明記する |
| 成果物だけ作って次のアクションが不明 | 担当者、期限、確認先を最後に整理する |

## 品質基準

- 初心者の RA が読んでも作業の順番を再現できること
- 制度・ルールの根拠と推測を区別していること
- 所属機関によって異なる事項には確認先を添えること
- 最終成果物に担当者、期限、次アクションが含まれること

## Available Tools

- **Read**: 公募要領、規程、既存資料の読み込み
- **Write**: 成果物、チェックリスト、メール文面の作成
- **WebSearch** / **WebFetch**: 公式情報、最新マニュアル、制度変更の確認
- **Bash**: ファイル整理、表の整形、文字数確認
