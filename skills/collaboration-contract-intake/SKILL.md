# 共同研究・契約インテーク

## What This Does

共同研究、受託研究、NDA、MTA、データ提供、再委託、覚書などの契約相談について、法務・契約・産学連携部署に渡す前の情報を整理する。契約条件を判断するスキルではなく、背景、相手先、研究内容、資金、成果物、知財、データ、期限、未決事項を漏れなく揃えるためのスキル。

- **対象ユーザー**: URA、産学連携担当、研究支援担当者、PI、部局事務
- **解決する課題**: 契約担当へ相談する材料が不足し、確認の往復で開始が遅れる
- **期待される効果**: 契約種別の取り違え、知財・データ・再委託の確認漏れ、相手先情報不足を減らす

## Required Inputs

- **相談目的**: NDA、共同研究、受託、データ提供、MTA、再委託、MOUなど
- **相手先情報**: 機関名、国、担当者、企業/大学/公的機関、反社・制裁等の確認状況
- **研究内容**: 目的、作業範囲、成果物、期間、場所、関係者
- **資金・物品・データ**: 金額、支払い、試料、機器、データ、秘密情報
- **知財・公開**: 発明可能性、論文発表、共同出願、成果公開の希望
- **期限**: 開始希望日、相手先回答期限、学内締切
- **既存資料**（任意）: 契約案、メール、研究計画、予算表、相手先様式

## What It Produces

1. **契約相談インテーク票**: 契約担当へ渡す基本情報
2. **論点整理表**: 契約、知財、データ、秘密情報、支払い、成果公開、再委託の未決事項
3. **確認依頼文**: 契約・法務・産学連携部署への相談メール
4. **相手先への追加確認リスト**: 先方に確認すべき不足情報
5. **開始前リスクメモ**: 契約前に開始してはいけない作業、要注意事項

## 手順

### 1. 契約相談の目的を確定する

契約名を断定せず、「何をしたいのか」から整理する。

| 目的 | 典型的に関係する契約 | 確認先 |
|------|----------------------|--------|
| 秘密情報を開示する | NDA | 契約・法務 |
| 共同で研究する | 共同研究契約 | 産学連携・契約 |
| 相手から業務を受ける | 受託研究契約 | 産学連携・経理 |
| データを受け渡す | データ提供契約、DUA | 契約・情報管理 |
| 試料を受け渡す | MTA | 契約・安全・倫理 |
| 一部業務を外注する | 再委託契約 | 契約・経理 |

### 2. 契約相談インテーク票を作る

```markdown
## 契約相談インテーク

- 相談目的:
- 相手先:
- 研究・業務内容:
- 期間:
- 金額・支払:
- 試料・データ・秘密情報:
- 成果物:
- 論文・広報・公開予定:
- 知財の可能性:
- 開始希望日:
- 添付資料:
```

### 3. 論点を分離する

```markdown
| 論点 | 現状 | 未決事項 | 確認先 | 期限 | 影響 |
|------|------|----------|--------|------|------|
```

最低限見る論点:

- 契約締結前に作業・情報開示・試料送付を始めていないか
- 成果物、検収、支払条件が曖昧でないか
- 発明・著作権・データ権利の扱いが未確認でないか
- 論文発表、プレス、学会発表の事前確認条項があるか
- 個人情報、機微情報、安全管理、輸出管理に触れないか

### 4. 専門部署への確認依頼を作る

```markdown
件名: [契約相談] {相手先名} との {相談目的} について

下記案件について、契約上の確認をお願いいたします。

概要:
- 相手先:
- 目的:
- 開始希望日:
- 添付資料:

確認したい事項:
- {論点1}
- {論点2}

未確認事項:
- {不足情報}
```

### 5. 相手先への追加確認を整理する

専門部署へ相談する前に不足が明らかな場合は、相手先へ短く確認する。

## 判断基準

| 判断項目 | 見るポイント | 対応 |
|----------|--------------|------|
| 開始前条件 | 契約前に着手してよいか | 契約担当に確認 |
| 秘密情報 | 何が秘密情報か | NDA要否を確認 |
| 知財 | 発明・成果物の帰属が絡むか | 知財担当へ確認 |
| データ・試料 | 個人情報、安全、輸出管理が絡むか | 専門部署へ分岐 |
| 期限 | 開始希望日に間に合うか | 逆算ToDoを作る |

## よくあるミスと対策

| ミス | 対策 |
|------|------|
| 契約名だけで相談する | 目的、作業範囲、成果物、相手先を整理する |
| 契約前に秘密情報を送る | NDA要否を先に確認する |
| 知財・論文公開を契約後まで見ない | 発明可能性と公開予定を最初に書く |
| 相手先契約案をそのまま進める | 所属機関の契約担当にレビュー依頼する |

## 品質基準

- 契約判断をRAが代行せず、確認先へ渡す材料になっていること
- 契約、知財、データ、倫理、安全、輸出管理を混ぜずに分岐できること
- 相手先、作業範囲、期間、金額、成果物、公開予定が整理されていること
- 契約前に止めるべき作業が明示されていること

## Available Tools

- **Read**: 契約案、メール、研究計画、予算表、相手先資料の読み込み
- **Write**: インテーク票、論点整理表、確認依頼文、追加確認リストの作成
- **WebSearch** / **WebFetch**: 相手先公開情報、所属機関公開規程の確認
- **Bash**: 表、チェックリスト、期限表の整形
